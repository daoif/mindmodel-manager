# 📖 MindModel Manager 使用指南

欢迎使用 MindModel Manager！本指南将帮助您快速上手，管理和运用您的思维模型库。

## 🚀 快速开始

### 1. 启动应用
- **首次运行**: 双击 `MindModelLauncher.exe`。它会自动配置运行环境。
- **日常使用**: 推荐直接运行 `MindModel.exe`，启动速度更快。

### 2. 创建第一个模型
1. 在主界面右上角，点击 **“新建模型”** 按钮。
   > 注意：如果您当前正在浏览某个特定分类，新建按钮可能隐藏，请先点击左侧侧边栏顶部的“全部模型”。
2. **填写基本信息**:
   - **名称**: 模型的核心名称（如“第一性原理”）。
   - **描述**: 简要的一句话介绍。
3. **打标签**: 为了方便后续检索，请为模型添加维度标签（见下文“核心概念”）。
4. 点击 **“保存”**。

---

## 🧩 核心概念：维度 (Dimensions)

我们的知识库通过四个核心维度来组织所有模型。理解这些维度有助于您更高效地检索和应用模型。

| 维度 | 说明 | 示例 |
| :--- | :--- | :--- |
| **场景 (Scene)** | 该模型适用的宏观领域 | 研发(Dev)、商业(Biz)、直播(Live)、个人成长(Self) |
| **阶段 (Stage)** | 在解决问题的哪个环节使用 | 洞察(Insight)、决策(Decision)、执行(Execution) |
| **类型 (Type)** | 该模型的用途属性 | 学习型(Coaching)、内化型(Internalization)、产出型(Output) |
| **分类 (Category)** | 学科或知识门类 | 心理学、战略、管理学、思维科学 |

---

## 📄 核心概念：文档类型 (Doc Types)

每个模型不仅仅是一个标题，它包含了一组标准化的子文档，覆盖从“理解”到“应用”的全过程。

1. **介绍 (Introduction)**
   - 模型的基本定义、来源和核心逻辑。
2. **使用场景 (Usage Scenario)**
   -这个模型适合解决什么问题？在什么情况下使用？
3. **陪练提示词 (Coaching Prompts)**
   - **用途**: 用于让 AI 扮演教练，通过提问引导**您**掌握这个思维模型。
   - **目标**: 帮助人学会模型。
4. **内化提示词 (Internalization Prompts)**
   - **用途**: 发送给 AI，让 **AI** 理解这个模型的逻辑。
   - **目标**: 它是其他 Prompt 的前置基础（Few-Shot 样本）。
5. **产出提示词 (Output Prompts)**
   - **用途**: 让已经“内化”了该模型的 AI，根据您的具体问题输出解决方案。
   - **目标**: 直接解决具体问题。

*在卡片列表页，您可以直接点击对应的按钮，一键复制这些提示词内容。*

## 🔍 检索与筛选

MindModel Manager 提供了强大的多维度检索功能，帮助您在知识海洋中快速定位。

### 1. 侧边栏导航
- 点击左侧的维度分类（如“场景 > Dev”），可以快速筛选出该维度下的所有模型。

### 2. 关键词搜索
- 在顶部搜索框输入关键词，支持搜索 **模型名称** 和 **描述**。

### 3. 高级筛选（多选）
- 在顶部筛选条点击具体维度（如“阶段”），您可以勾选多个标签。
- **逻辑**:
  - 不同维度之间是 **AND** 关系（例如：既是“Dev”场景 **且** 是“执行”阶段）。
  - 同一维度内部是 **OR** 关系（例如：查找“洞察” **或** “决策”阶段的模型）。

### 4. 排序与视图
- **排序**: 支持按“最新更新”、“名称”或“分类”排序。
- **视图**: 支持 **卡片视图**（信息密度低，适合浏览）和 **列表视图**（信息密度高，适合管理）。
